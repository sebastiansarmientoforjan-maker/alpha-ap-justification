# Week 1: Sistema Dinámico de Generación de Problemas

## 🎯 Concepto Clave

**Week 1 NO es un set fijo de 3 problemas.**

Week 1 es una **introducción general a error-forcing pedagogy** donde los problemas son **generados dinámicamente por Claude** basados en:
1. Score del quiz de MathAcademy (< 80% o ≥ 80%)
2. Tópicos donde el estudiante falló (si < 80%)
3. Tipo de FRQ asignado (específico vs general)

## 🤖 Claude Prompt Template para Generación Dinámica

```
You are an AP Mathematics FRQ generator. Generate ONE Free-Response Question and THREE Week 1 training problems for a student.

STUDENT CONTEXT:
- Name: {studentName}
- Course: {course} (AP Calculus BC)
- Quiz Score: {score}% on {quizTopic}
- Weak Topics: {weakTopics} (e.g., ["derivatives", "MVT conditions"])
- Reasoning Stage: {reasoningStage} (empirical/generic/formal)

FRQ TYPE: {frqType}
- If "specific": Target the weak topics listed above
- If "general": Mix topics across the curriculum

WEEK 1 FOCUS:
Generate 3 error-forcing problems that:
1. Look solvable at first glance
2. Break if theorem conditions are skipped
3. Match the student's reasoning stage (provide appropriate scaffolding)
4. Target weak topics if FRQ type is "specific"

ERROR CATEGORIES:
- CONDITION_BYPASS: Student skips verifying theorem hypotheses
- LOCAL_ONLY_ARGUMENT: Student checks one condition, misses another
- CER_BREAKDOWN: Student states theorem but doesn't connect evidence

OUTPUT FORMAT (JSON):
{
  "frq": {
    "id": "frq-{uuid}",
    "statement": "...",
    "type": "specific" or "general",
    "targetedTopics": [...],
    "scaffoldingLevel": "full" | "structural" | "minimal",
    "rubric": {...},
  },
  "week1Problems": [
    {
      "id": "w1-gen-{uuid}",
      "title": "...",
      "statement": "...", // LaTeX formatted
      "course": "calculus-bc",
      "errorCategory": "CONDITION_BYPASS",
      "trap": "...",
      "correctCERCResponse": {...},
      "sentenceFrames": {...},
      "theoremInfo": {...},
      "hints": {
        "level1": "...",
        "level2": "...",
        "level3": "..."
      }
    },
    // Problem 2
    // Problem 3
  ]
}

IMPORTANT:
- Problems must be unique to this student's needs
- Scaffolding must match reasoning stage
- Traps must be subtle but pedagogically sound
- All problems must use CERC framework
```

## 🔧 Implementación Técnica

### API Endpoint: Generate FRQ + Week 1 Problems
```typescript
// app/api/admin/generate-frq/route.ts

export async function POST(req: Request) {
  const { studentId, quizScore, weakTopics, course } = await req.json();

  // Step 1: Decide FRQ type
  const frqType = quizScore < 80 ? "specific" : "general";

  // Step 2: Get student reasoning stage (from pre-assessment or history)
  const reasoningStage = await getStudentReasoningStage(studentId);

  // Step 3: Call Claude API
  const generation = await anthropic.messages.create({
    model: "claude-opus-4-6",
    max_tokens: 4000,
    system: WEEK_1_GENERATION_PROMPT, // From above
    messages: [{
      role: "user",
      content: JSON.stringify({
        studentId,
        course,
        quizScore,
        weakTopics,
        frqType,
        reasoningStage,
      }),
    }],
  });

  const result = JSON.parse(generation.content[0].text);

  // Step 4: Save to database (pending approval)
  await db.collection("frq_pending_approval").add({
    ...result,
    studentId,
    status: "pending",
    createdAt: new Date(),
  });

  // Step 5: Notify bot
  await notifyBot({
    type: "frq_approval_needed",
    studentId,
    frqId: result.frq.id,
    score: quizScore,
    topics: weakTopics,
  });

  return Response.json({
    success: true,
    frqId: result.frq.id,
    message: "FRQ generated and sent for approval",
  });
}
```

### Admin Approval Dashboard
```typescript
// app/admin/frq-approvals/page.tsx

export default function FRQApprovalsPage() {
  const pendingFRQs = usePendingFRQs(); // Real-time listener

  return (
    <div>
      <h1>FRQ Approvals Needed</h1>
      {pendingFRQs.map(frq => (
        <FRQApprovalCard
          key={frq.id}
          frq={frq}
          onApprove={async () => {
            // Approve and assign to student
            await approveFRQ(frq.id);
            await assignToStudent(frq.studentId, frq.id);
          }}
          onRegenerate={async () => {
            // Call Claude API again with feedback
            await regenerateFRQ(frq.id, "Make trap more subtle");
          }}
          onEdit={(editedContent) => {
            // Sebastian manually edits FRQ
            await updateFRQ(frq.id, editedContent);
          }}
        />
      ))}
    </div>
  );
}
```
